Overview

The Credit Scoring Model is a machine learning project developed to predict an individual's creditworthiness using historical financial information. The system analyzes customer financial records and classifies applicants as either low risk or high risk. This helps financial institutions make informed lending decisions and reduce the possibility of loan defaults. The project applies advanced machine learning techniques including data preprocessing, feature engineering, class balancing, feature selection, hyperparameter tuning, and model optimization to achieve reliable predictive performance.

Objectives

The primary objective of this project is to build an accurate and robust classification model capable of evaluating customer credit risk based on financial attributes. The model is designed to identify potentially risky applicants while maintaining balanced performance across multiple evaluation metrics.

Project Workflow

Data Preprocessing
The dataset is loaded and cleaned before model training. Duplicate records are removed and invalid target values are excluded. The target variable is converted into a binary classification format suitable for machine learning algorithms.

Feature Engineering
Several financial indicators are created to enhance predictive capability.

Generated features include:
Debt Income Ratio
Loan Income Ratio
Expense Income Ratio
Debt Asset Ratio
Savings Estimate
Net Worth

These derived features provide additional insights into customer financial behavior and improve model performance.

Data Transformation
Categorical variables are converted into numerical representations using one hot encoding. This enables the machine learning model to process all features effectively.

Handling Class Imbalance
The dataset contains an unequal distribution of credit classes. Synthetic Minority Oversampling Technique (SMOTE) is applied to generate synthetic samples for the minority class and improve learning performance.

Hyperparameter Optimization
Randomized Search Cross Validation is used to identify the most effective XGBoost parameters. Multiple parameter combinations are evaluated using stratified cross validation and ROC AUC scoring.

Feature Selection
Feature importance scores generated by XGBoost are used to select the most informative variables. This reduces noise, improves model efficiency, and enhances generalization.

Model Training
The final model is trained using XGBoost Classifier with optimized hyperparameters and selected features. XGBoost was chosen because of its strong performance on structured tabular datasets and its ability to handle complex feature relationships.

Model Evaluation
The trained model is evaluated using several classification metrics.

Accuracy
Measures the overall percentage of correct predictions.

Precision
Measures how many applicants predicted as high risk are actually high risk.

Recall
Measures how many actual high risk applicants are correctly identified by the model.

F1 Score
Provides a balanced evaluation of Precision and Recall.

ROC AUC
Measures the model's ability to distinguish between good and bad credit applicants across different classification thresholds.

Results
The optimized model achieved strong performance across multiple evaluation metrics.
Accuracy: 78.90%
Precision: 59.63%
Recall: 64.00%
F1 Score: 61.74%
ROC AUC: 82.87%
These results demonstrate that the model can effectively identify risky applicants while maintaining balanced overall performance.
